# Dataset Information

## Dataset Used

**Name:** SIDD — Smartphone Image Denoising Dataset

**Paper:** Abdelrahman Abdelhamed, Stephen Lin, Michael S. Brown. "A High-Quality Denoising Dataset for Smartphone Cameras." CVPR 2018.

**Download:** https://www.kaggle.com/datasets/gowthamrev/side-full

## What SIDD Contains

SIDD contains pairs of noisy and clean images taken on 5 different smartphone cameras under different lighting conditions. The noisy images use the phone's maximum ISO setting to capture real sensor noise. The clean images are generated by taking many photos of the same scene and averaging them to remove noise.

This makes it a good dataset for training denoising models because the noise is real sensor noise, not synthetic Gaussian noise added artificially.

## Why SIDD Was Chosen

- It has real smartphone noise, which matches the personal smartphone images used for testing
- It is a well-known benchmark, so results can be compared to published papers
- It is large enough for training (30,000 patches were extracted)
- Using only one dataset avoids domain mixing problems that caused instability in earlier training attempts

## How the Data Was Used

The SIDD images were not used directly. Instead, 256x256 pixel patches were extracted randomly from the full images.

- Training: 30,000 patches with random flips and rotations for augmentation
- Validation: 1,000 patches with no augmentation

The full dataset is not included in this repository because it is around 12 GB. It is only needed for retraining. For inference, only the checkpoint is needed.

## Real Test Images

The final model was tested on photos taken with a personal smartphone (Vivo X200 Pro). A selection of comparison results is included in `results/samples/`.
